Textformatierung

CSS bietet viele Möglichkeiten, Text zu gestalten.

Text zentrieren (text-align)

Mit text-align: center; kannst du Text horizontal zentrieren.

Dieser Text ist zentriert.

p { text-align: center; }

Weitere Werte für text-align sind left (linksbündig, Standard), right (rechtsbündig) und justify (Blocksatz).

Farben und Hintergründe

Textfarbe (color)

Mit color änderst du die Farbe des Textes.

Dieser Text ist grün.

p { color: green; /* Weitere Möglichkeiten: #008000 (Hex), rgb(0,128,0) */ }

Hintergrundfarbe (background-color)

Mit background-color setzt du die Hintergrundfarbe eines Elements.

Dieser Absatz hat einen hellblauen Hintergrund.

p { background-color: lightblue; /* Weitere Möglichkeiten: #add8e6 (Hex), rgb(173,216,230) */ }

Hintergrundbild (background-image)

Mit background-image: url('pfad/zum/bild.jpg'); kannst du ein Hintergrundbild festlegen.

Zusätzliche Eigenschaften sind:

  • background-repeat: (no-repeat, repeat-x, repeat-y, repeat)
  • background-position: (z.B. center, top left, 50% 50%)
  • background-size: (z.B. cover, contain, 100px 50px)
  • background-attachment: (scroll, fixed)
div { background-image: url('pfad/zum/bild.jpg'); background-repeat: no-repeat; background-position: center; background-size: cover; }

Schriftarten (Fonts)

Schriftfamilie (font-family)

Bestimmt die Schriftart des Textes. Es ist üblich, mehrere Schriftarten als Fallback anzugeben.

Dieser Text verwendet Georgia oder eine Serif-Schriftart.

p { font-family: 'Georgia', serif; }

Schriftgröße (font-size)

Legt die Größe der Schrift fest. Einheiten können Pixel (px), Punkte (pt), em, rem, Prozent (%) sein.

Dieser Text ist 24 Pixel groß.

p { font-size: 24px; }

Schriftgewicht (font-weight)

Bestimmt die Dicke der Schrift (z.B. normal, fett).

Dieser Text ist fett.

p { font-weight: bold; /* auch: normal, lighter, bolder, 100-900 */ }

Weitere Schrift-Eigenschaften

  • font-style: z.B. italic (kursiv), normal.
  • line-height: Zeilenhöhe.
  • text-decoration: z.B. underline (unterstrichen), none, line-through.
  • text-transform: z.B. uppercase (Großbuchstaben), lowercase, capitalize.
  • letter-spacing: Abstand zwischen Buchstaben.
  • word-spacing: Abstand zwischen Wörtern.

Das Boxmodell

Jedes HTML-Element kann als eine rechteckige Box betrachtet werden. Das Boxmodell beschreibt, wie diese Boxen aufgebaut sind:

  • Content: Der Inhalt des Elements (Text, Bilder etc.).
  • Padding: Der innere Abstand zwischen dem Content und dem Border.
  • Border: Der Rahmen um das Padding und den Content.
  • Margin: Der äußere Abstand um den Border, der das Element von anderen Elementen trennt.
Dies ist der Inhalt (Content).
.box-model-example { width: 300px; /* Breite des Contents */ padding: 20px; /* Innenabstand auf allen Seiten */ border: 5px solid navy; /* Rahmen */ margin: 15px; /* Außenabstand auf allen Seiten */ background-color: lightgray; /* Hintergrund des Contents + Paddings */ }

Die Gesamtbreite eines Elements ist also: width + padding-left + padding-right + border-left + border-right.

Mit box-sizing: border-box; kann man das Verhalten ändern, sodass width die Gesamtbreite inklusive Padding und Border definiert.

* { /* Gilt für alle Elemente */ box-sizing: border-box; }

Margin

margin erzeugt einen Außenabstand um ein Element.

Dieses Element hat 20px Margin außen.
.margin-example { margin: 20px; /* Alle Seiten */ /* margin-top: 10px; margin-right: 5px; margin-bottom: 10px; margin-left: 5px; */ /* margin: 10px 5px; (oben/unten rechts/links) */ /* margin: 10px 5px 15px; (oben rechts/links unten) */ /* margin: 10px 5px 15px 20px; (oben rechts unten links) */ }

Padding

padding erzeugt einen Innenabstand innerhalb eines Elements, zwischen Inhalt und Rahmen.

Dieses Element hat 30px Padding innen.
.padding-example { padding: 30px; /* Alle Seiten */ /* padding-top, padding-right, padding-bottom, padding-left analog zu margin */ }

Border

border definiert einen Rahmen um ein Element.

Dieses Element hat einen gestrichelten orangen Rahmen.
.border-example { border: 3px dashed orange; /* border-width: 3px; */ /* border-style: dashed; (solid, dotted, double, groove, ridge, inset, outset) */ /* border-color: orange; */ /* Auch: border-top, border-right, border-bottom, border-left */ border-radius: 10px; /* Abgerundete Ecken */ }

Breite und Höhe (width, height)

Steuert die Breite und Höhe des Content-Bereichs eines Elements (außer bei box-sizing: border-box).

200x100 Box
.width-height-example { width: 200px; height: 100px; } /* Weitere Eigenschaften: min-width, max-width, min-height, max-height */

Anzeigetyp (display)

Die Eigenschaft display legt fest, wie ein Element dargestellt wird und wie es sich im Layout verhält.

display: inline;

Inline-Elemente nehmen nur so viel Breite ein, wie ihr Inhalt benötigt und erlauben keine explizite Höhen- und Breitensetzung. Sie fließen im Text mit und erzeugen keinen Zeilenumbruch. Beispiele: <span>, <a>, <img>.

Dies ist ein Span (inline). Ein weiteres Span. Ein Link (inline).
span { display: inline; }

display: block;

Block-Elemente nehmen standardmäßig die volle verfügbare Breite ein und beginnen immer in einer neuen Zeile. Höhe und Breite können explizit gesetzt werden. Beispiele: <div>, <p>, <h1>-<h6>, <ul>.

Dies ist ein Div (block).
Ein weiteres Div.

Ein Paragraph (block).

div { display: block; }

display: inline-block;

Eine Mischung aus Inline und Block: Das Element fließt wie ein Inline-Element im Text mit, aber man kann Höhe, Breite, Margin und Padding wie bei einem Block-Element setzen.

Box 1
Box 2
Box 3
div { display: inline-block; width: 100px; height: 50px; }

display: none;

Das Element wird komplett ausgeblendet und nimmt keinen Platz im Layout ein.

Weitere Werte für display

Es gibt viele weitere Werte, darunter flex und grid, die für komplexe Layouts verwendet werden (siehe separater Abschnitt).

Positionierung (position)

Die position-Eigenschaft bestimmt, wie ein Element im Dokument positioniert wird. In Kombination mit den Eigenschaften top, right, bottom und left kann die genaue Position festgelegt werden.

position: static; (Standard)

Elemente werden normal im Fluss des Dokuments angeordnet. Die Eigenschaften top, right, bottom, left und z-index haben keine Auswirkung.

position: relative;

Das Element wird normal im Fluss des Dokuments positioniert. Mit top, right, bottom, left kann es relativ zu seiner normalen Position verschoben werden. Der ursprüngliche Platz des Elements bleibt reserviert.

Ich bin relativ positioniert.
Inneres Element, relativ zum Äußeren verschoben.
.parent { position: relative; } .child { position: relative; /* oder absolute, fixed */ top: 10px; left: 20px; }

position: absolute;

Das Element wird aus dem normalen Dokumentfluss entfernt. Es wird relativ zum nächsten positionierten Elternelement (ein Element mit position außer static) positioniert. Wenn kein solches Elternelement existiert, wird es relativ zum initialen Containerblock (meistens der <body>) positioniert.

Bezugspunkt (relative Position)
Ich bin absolut positioniert.
.parent { position: relative; /* Wichtig als Bezugspunkt! */ } .child { position: absolute; top: 10px; right: 10px; }

position: fixed;

Ähnlich wie absolute, aber das Element wird relativ zum Viewport (dem sichtbaren Fensterbereich des Browsers) positioniert. Es bleibt an seiner Position, auch wenn die Seite gescrollt wird. Ein Beispiel ist oben rechts auf dieser Seite zu sehen.

.fixed-element { position: fixed; bottom: 20px; right: 20px; }

position: sticky;

Eine Mischung aus relative und fixed. Das Element verhält sich zunächst wie relative, bis ein bestimmter Scroll-Punkt (Offset, z.B. top: 0) erreicht wird. Dann verhält es sich wie fixed. Ein Beispiel ist der orange Balken oben auf dieser Seite, der beim Scrollen oben "kleben" bleibt.

.sticky-element { position: sticky; top: 0; /* Bleibt am oberen Rand kleben, wenn gescrollt wird */ }

float: left; oder float: right; lässt Elemente nach links oder rechts "fließen", wobei der umgebende Inhalt sie umfließt. Früher oft für Spaltenlayouts genutzt, heute eher für das Umfließen von Bildern durch Text.

Ich floate links.

.float-left { float: left; margin-right: 10px; } .clear-fix::after { /* Behebt Probleme mit kollabierenden Elternelementen */ content: ""; clear: both; display: table; }

Layout (Flexbox & Grid)

Moderne CSS-Layouttechniken für flexible und komplexe Anordnungen von Elementen.

Flexbox (display: flex;)

Flexbox ist ein eindimensionales Layoutmodell, das Elemente in einer Zeile oder Spalte anordnen kann. Es eignet sich hervorragend für Komponenten und kleinere Layoutbereiche.

Eigenschaften für den Flex-Container:

  • display: flex;: Aktiviert Flexbox.
  • flex-direction: Hauptachse (row, row-reverse, column, column-reverse).
  • justify-content: Ausrichtung der Items entlang der Hauptachse (z.B. flex-start, flex-end, center, space-between, space-around, space-evenly).
  • align-items: Ausrichtung der Items entlang der Querachse (z.B. stretch, flex-start, flex-end, center, baseline).
  • flex-wrap: Umbruchverhalten (nowrap, wrap, wrap-reverse).
  • align-content: Ausrichtung der Zeilen bei mehrzeiligem Flexbox (wenn flex-wrap: wrap;).
  • gap: Abstand zwischen Flex-Items.

Item 2
Item 3
.flex-container { display: flex; justify-content: space-around; /* Verteilt Items mit gleichem Abstand */ align-items: center; /* Zentriert Items vertikal */ background-color: dodgerblue; } .flex-item { background-color: #f1f1f1; padding: 20px; }

Eigenschaften für Flex-Items:

  • order: Reihenfolge der Items.
  • flex-grow: Faktor, um den das Item wachsen darf.
  • flex-shrink: Faktor, um den das Item schrumpfen darf.
  • flex-basis: Basisgröße des Items.
  • flex: Kurzform für flex-grow, flex-shrink und flex-basis.
  • align-self: Überschreibt align-items für ein einzelnes Item.

Grid (display: grid;)

CSS Grid ist ein zweidimensionales Layoutmodell, das Inhalte in Zeilen und Spalten anordnet. Es ist ideal für komplexe Seitenlayouts.

Eigenschaften für den Grid-Container:

  • display: grid;: Aktiviert Grid.
  • grid-template-columns: Definiert die Spalten (z.B. 100px 1fr 2fr;).
  • grid-template-rows: Definiert die Zeilen (z.B. auto 100px;).
  • grid-template-areas: Definiert benannte Grid-Bereiche.
  • gap (oder grid-gap, row-gap, column-gap): Abstand zwischen Grid-Zellen.
  • justify-items: Ausrichtung der Items innerhalb ihrer Zelle entlang der Inline-Achse (horizontal).
  • align-items: Ausrichtung der Items innerhalb ihrer Zelle entlang der Block-Achse (vertikal).
  • justify-content: Ausrichtung des gesamten Grids im Container (wenn das Grid kleiner ist als der Container).
  • align-content: Ausrichtung des gesamten Grids im Container (wenn das Grid kleiner ist als der Container).

Eigenschaften für Grid-Items:

  • grid-column-start, grid-column-end, grid-column: Platzierung in Spalten.
  • grid-row-start, grid-row-end, grid-row: Platzierung in Zeilen.
  • grid-area: Benennt ein Item für grid-template-areas oder Kurzform für Zeilen/Spalten.
  • justify-self: Überschreibt justify-items für ein einzelnes Item.
  • align-self: Überschreibt align-items für ein einzelnes Item.

Pseudo-Klassen und Pseudo-Elemente

Ermöglichen das Stylen von Elementen basierend auf ihrem Zustand oder das Einfügen von Inhalten.

Pseudo-Klassen

Stylen Elemente basierend auf ihrem Zustand oder ihrer Position im Dokumentbaum. Beginnen mit einem Doppelpunkt (:).

  • :hover: Element, über dem sich der Mauszeiger befindet.
  • :active: Element, das gerade geklickt wird.
  • :focus: Element, das den Fokus hat (z.B. ein Eingabefeld).
  • :link: Noch nicht besuchter Link.
  • :visited: Bereits besuchter Link.
  • :first-child: Erstes Kindelement eines Elternelements.
  • :last-child: Letztes Kindelement.
  • :nth-child(n): Kindelement an n-ter Position (z.B. :nth-child(2n) für jedes zweite Element, :nth-child(odd) für ungerade).
  • :disabled: Deaktiviertes Formularelement.
  • :checked: Aktivierte Checkbox oder Radio-Button.
  • :empty: Element ohne Kindelemente oder Textinhalt.
  • :not(selector): Elemente, die nicht dem Selektor entsprechen.
Test Link
a:link { color: blue; } /* Unbesuchter Link */ a:visited { color: purple; } /* Besuchter Link */ a:hover { color: red; } /* Link bei Mouseover */ a:active { color: orange; } /* Link während Klick */ input:focus { border-color: blue; } /* Eingabefeld mit Fokus */

Pseudo-Elemente

Stylen bestimmte Teile eines Elements oder fügen generierten Inhalt hinzu. Beginnen mit zwei Doppelpunkten (::), obwohl aus Kompatibilitätsgründen oft auch ein einzelner Doppelpunkt funktioniert.

  • ::before: Fügt Inhalt vor dem Inhalt des Elements ein.
  • ::after: Fügt Inhalt nach dem Inhalt des Elements ein. (Wird oft für Clearfixes verwendet).
  • ::first-line: Stylt die erste Zeile eines Blockelements.
  • ::first-letter: Stylt den ersten Buchstaben eines Blockelements.
  • ::selection: Stylt den vom Benutzer markierten Text.
  • ::placeholder: Stylt den Platzhaltertext in Formularfeldern.

Die erste Zeile dieses Absatzes wird speziell gestylt. Der Rest des Textes bleibt normal. Lorem ipsum dolor sit amet.

p::first-line { color: #ff0000; font-variant: small-caps; }

Der erste Buchstabe wird speziell gestylt. Lorem ipsum dolor sit amet.

p::first-letter { color: #0000ff; font-size: xx-large; }
Inhalt der Box
.box::before { content: "Anfang: "; color: green; } .box::after { content: " :Ende"; color: red; }

Effekte, Transformationen & Animationen

Deckkraft (opacity)

Steuert die Transparenz eines Elements. Wert von 0 (komplett transparent) bis 1 (komplett sichtbar).

Bewege die Maus über das Bild.

img { opacity: 0.5; } img:hover { opacity: 1.0; }

Schatten (box-shadow, text-shadow)

box-shadow fügt einen Schatteneffekt zu Elementboxen hinzu.

Schatten
div { box-shadow: H-Offset V-Offset Weichzeichner Spreizung Farbe [inset]; /* Beispiel: box-shadow: 5px 5px 10px #888; */ }

text-shadow fügt einen Schatteneffekt zu Text hinzu.

Text mit Schatten

p { text-shadow: H-Offset V-Offset Weichzeichner Farbe; /* Beispiel: text-shadow: 2px 2px 4px #555; */ }

Überlauf (overflow)

Steuert, wie Inhalt behandelt wird, der die Dimensionen seines Containers überschreitet.

  • visible (Standard): Inhalt wird außerhalb der Box angezeigt.
  • hidden: Überlaufender Inhalt wird abgeschnitten und ist nicht sichtbar.
  • scroll: Scrollbalken werden immer angezeigt, auch wenn nicht nötig.
  • auto: Scrollbalken werden nur angezeigt, wenn der Inhalt überläuft.

Stapelanordnung (z-index)

Gibt die Stapelreihenfolge von positionierten Elementen an (Elemente mit position außer static). Elemente mit höherem z-index liegen über Elementen mit niedrigerem z-index.

Box 1 (z-index: 1)
Box 2 (z-index: 2)
Box 3 (z-index: 0)
.box1 { position: absolute; z-index: 1; background: red; } .box2 { position: absolute; z-index: 2; background: green; top: 30px; left: 30px; } /* Liegt über Box 1 */ .box3 { position: absolute; z-index: 0; background: blue; top: 60px; left: 60px; } /* Liegt unter Box 1 */

Transformationen (transform)

Ermöglicht das Drehen, Skalieren, Verschieben oder Neigen von Elementen.

  • translate(x, y): Verschiebt das Element. (z.B. transform: translateX(50px);)
  • rotate(angle): Dreht das Element. (z.B. transform: rotate(45deg);)
  • scale(x, y): Skaliert das Element. (z.B. transform: scale(1.5);)
  • skew(x-angle, y-angle): Neigt das Element. (z.B. transform: skewX(20deg);)
  • matrix(...): Kombiniert alle Transformationen.
Transformiert
div { transform: rotate(15deg) translateX(20px) scale(1.2); } /* Um den Ursprung der Transformation zu ändern: transform-origin: center; (Standard) */

Übergänge (transition)

Ermöglicht sanfte Animationen von CSS-Eigenschaftsänderungen über eine bestimmte Zeit.

Wichtige Eigenschaften:

  • transition-property: Die CSS-Eigenschaft(en), die animiert werden sollen (z.B. width, background-color, all).
  • transition-duration: Die Dauer des Übergangs (z.B. 2s, 500ms).
  • transition-timing-function: Die Beschleunigungskurve des Übergangs (z.B. ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier(...)).
  • transition-delay: Eine Verzögerung, bevor der Übergang startet.
  • transition: Kurzform für alle vier Eigenschaften.
Bewege die Maus hier rüber!
div { width: 100px; height: 100px; background: red; transition: width 2s, height 2s, transform 2s; /* Kurzform: transition: all 2s ease-in-out 0.5s; */ } div:hover { width: 200px; height: 200px; transform: rotate(180deg); }

Animationen (@keyframes und animation)

Erlauben komplexere, mehrstufige Animationen.

@keyframes definiert die einzelnen Schritte (Keyframes) einer Animation.

Die animation-Eigenschaft (und ihre Untereigenschaften) wendet die Keyframe-Animation auf ein Element an:

  • animation-name: Name der @keyframes-Regel.
  • animation-duration: Dauer der Animation.
  • animation-timing-function: Beschleunigungskurve.
  • animation-delay: Verzögerung.
  • animation-iteration-count: Wie oft die Animation abgespielt wird (z.B. infinite).
  • animation-direction: Richtung (normal, reverse, alternate, alternate-reverse).
  • animation-fill-mode: Welche Styles angewendet werden, bevor/nachdem die Animation läuft (none, forwards, backwards, both).
  • animation-play-state: Ob die Animation läuft oder pausiert ist (running, paused).
  • animation: Kurzform für alle Eigenschaften.
Ich animiere!
@keyframes exampleanimation { 0% {background-color:red; left:0px; top:0px; transform: scale(1) rotate(0deg);} 25% {background-color:yellow; left:100px; top:0px; transform: scale(1.2) rotate(90deg);} 50% {background-color:blue; left:100px; top:100px; transform: scale(1) rotate(180deg);} 75% {background-color:green; left:0px; top:100px; transform: scale(1.2) rotate(270deg);} 100% {background-color:red; left:0px; top:0px; transform: scale(1) rotate(360deg);} } div { width: 100px; height: 100px; position: relative; /* Nötig für left/top Animation */ animation-name: exampleanimation; animation-duration: 8s; animation-iteration-count: infinite; animation-direction: alternate; }

Responsive Webdesign & Media Queries

Responsive Webdesign zielt darauf ab, Webseiten so zu gestalten, dass sie auf verschiedenen Gerätegrößen (Desktops, Tablets, Smartphones) optimal dargestellt werden und benutzerfreundlich sind.

Ein wichtiger Baustein dafür sind Media Queries.

Meta Viewport Tag

Zuerst sollte im <head> der HTML-Datei der Viewport Meta-Tag gesetzt werden, um die Seite auf mobilen Geräten korrekt zu skalieren:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries (@media)

Mit Media Queries können CSS-Regeln definiert werden, die nur unter bestimmten Bedingungen gelten, z.B. abhängig von der Bildschirmbreite, -höhe, Ausrichtung (Hoch-/Querformat) oder Auflösung.

Syntax: @media Medientyp and (Bedingung) { /* CSS-Regeln */ }

  • Medientypen: all (alle Geräte), screen (Bildschirme), print (Druckansicht), speech (Screenreader).
  • Bedingungen (Media Features):
    • width, min-width, max-width (Breite des Viewports)
    • height, min-height, max-height (Höhe des Viewports)
    • orientation: portrait | landscape (Ausrichtung)
    • aspect-ratio, min-aspect-ratio, max-aspect-ratio (Seitenverhältnis)
    • resolution, min-resolution, max-resolution (Auflösung)
Dieser Bereich ändert sein Aussehen bei Bildschirmbreiten unter 768px. Verändere die Fenstergröße!
.responsive-example { width: 100%; background-color: lightcyan; } /* Wenn die Bildschirmbreite 768px oder weniger beträgt */ @media screen and (max-width: 768px) { .responsive-example { background-color: lightcoral; font-size: 14px; } /* Weitere Anpassungen für kleinere Bildschirme: */ nav ul li { display: block; /* Menüpunkte untereinander */ } .flex-container-example { flex-direction: column; /* Flex-Items untereinander */ } }

Mobile First Ansatz: Es ist oft sinnvoll, zuerst die Styles für mobile Geräte zu definieren und dann mit Media Queries (min-width) schrittweise Anpassungen für größere Bildschirme hinzuzufügen.
