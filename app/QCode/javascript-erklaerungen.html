Was ist JavaScript?

JavaScript (oft abgekürzt als JS) ist eine dynamische, interpretierte Skriptsprache, die hauptsächlich im Webbrowser verwendet wird, um Webseiten interaktiv zu gestalten. Es ist eine der Kerntechnologien des World Wide Web, neben HTML und CSS.

Mit JavaScript kannst du:

  • HTML-Inhalte und CSS-Stile dynamisch verändern.
  • Auf Benutzeraktionen reagieren (Klicks, Tastatureingaben, Mausbewegungen).
  • Daten im Hintergrund laden (AJAX).
  • Animationen erstellen.
  • Formulare validieren.
  • Und vieles mehr!

Obwohl es ursprünglich für Browser entwickelt wurde, kann JavaScript dank Umgebungen wie Node.js auch serverseitig (Backend) oder für Desktop- und Mobile-Anwendungen verwendet werden.

Wichtig: JavaScript ist nicht dasselbe wie Java! Es sind zwei unterschiedliche Programmiersprachen mit unterschiedlichen Anwendungsgebieten, auch wenn der Name ähnlich klingt.

Variablen und Datentypen

Variablen sind Container zum Speichern von Datenwerten.

Variablen deklarieren

In modernem JavaScript (ES6+) werden Variablen hauptsächlich mit let und const deklariert. var ist eine ältere Methode und sollte vermieden werden.

  • let: Deklariert eine Variable, deren Wert neu zugewiesen werden kann. Sie ist block-scoped (nur innerhalb des Blocks gültig, in dem sie definiert wurde).
  • const: Deklariert eine Konstante, deren Wert nach der ersten Zuweisung nicht mehr geändert werden kann. Auch block-scoped.
  • var: Ältere Deklaration. Ist funktions-scoped oder global-scoped und hat einige Eigenheiten (Hoisting), die zu Verwirrung führen können.
let alter = 30; alter = 31; // Ok const name = "Max"; // name = "Moritz"; // Fehler! Konstanten können nicht neu zugewiesen werden. var stadt = "Berlin"; // Ältere Schreibweise console.log(alter); // Gibt 31 aus console.log(name); // Gibt "Max" aus console.log(stadt); // Gibt "Berlin" aus

Datentypen

JavaScript ist eine dynamisch typisierte Sprache, d.h. der Datentyp einer Variablen wird zur Laufzeit bestimmt und kann sich ändern.

Primitive Datentypen:

  • String: Eine Zeichenkette, z.B. "Hallo Welt" oder 'JavaScript'.
  • Number: Eine Zahl, z.B. 42 oder 3.14. (Es gibt keine Unterscheidung zwischen Integer und Float).
  • Boolean: Ein Wahrheitswert, entweder true oder false.
  • Undefined: Eine Variable, die deklariert, aber nicht initialisiert wurde, hat den Wert undefined.
  • Null: Repräsentiert absichtlich das Nichtvorhandensein eines Objektwertes. Es ist ein zugewiesener Wert.
  • Symbol (ES6): Ein eindeutiger und unveränderlicher primitiver Wert, der als Schlüssel für Objekteigenschaften verwendet werden kann.
  • BigInt (ES2020): Für ganze Zahlen, die größer sind als der Bereich, den der Number-Typ sicher darstellen kann.

Komplexer Datentyp:

  • Object: Eine Sammlung von Schlüssel-Wert-Paaren. Arrays und Funktionen sind spezielle Arten von Objekten.
let text = "Hallo"; // String let zahl = 123; // Number let istWahr = true; // Boolean let nichtDefiniert; // Undefined (automatisch) let leer = null; // Null let person = { // Object vorname: "Anna", nachname: "Muster" }; let farben = ["rot", "grün", "blau"]; // Array (spezielles Objekt) console.log(typeof text); // "string" console.log(typeof zahl); // "number" console.log(typeof person); // "object"

Operatoren

Operatoren werden verwendet, um Operationen auf Werten und Variablen durchzuführen.

Arithmetische Operatoren

+ (Addition), - (Subtraktion), * (Multiplikation), / (Division), % (Modulo, Rest einer Division), ** (Exponentiation, ES7).

let a = 10; let b = 3; console.log(a + b); // 13 console.log(a % b); // 1 console.log(2 ** 3); // 8 (2 hoch 3)

Zuweisungsoperatoren

=, +=, -=, *=, /=, %=.

let x = 5; x += 3; // Äquivalent zu x = x + 3; (x ist jetzt 8)

Vergleichsoperatoren

== (gleich, mit Typumwandlung), === (strikt gleich, Wert und Typ müssen gleich sein), != (ungleich, mit Typumwandlung), !== (strikt ungleich), > (größer als), < (kleiner als), >= (größer oder gleich), <= (kleiner oder gleich).

Empfehlung: Verwende fast immer === und !==, um unerwartetes Verhalten durch automatische Typumwandlung zu vermeiden.

console.log(5 == "5"); // true (Typumwandlung) console.log(5 === "5"); // false (keine Typumwandlung) console.log(10 > 5); // true

Logische Operatoren

&& (logisches UND), || (logisches ODER), ! (logisches NICHT).

let istErwachsen = true; let hatTicket = false; console.log(istErwachsen && hatTicket); // false console.log(istErwachsen || hatTicket); // true console.log(!hatTicket); // true

Weitere Operatoren

  • Inkrement/Dekrement: ++, -- (z.B. x++ oder ++x).
  • Ternärer Operator (Bedingungsoperator): bedingung ? wert_wenn_wahr : wert_wenn_falsch.
  • typeof Operator: Gibt den Typ einer Variablen als String zurück.
let alter = 18; let status = (alter >= 18) ? "Volljährig" : "Minderjährig"; console.log(status); // "Volljährig" let wert = 42; console.log(typeof wert); // "number"

Kontrollstrukturen

Steuern den Ablauf eines Programms.

Bedingte Anweisungen (if, else if, else)

Führt Codeblöcke basierend auf Bedingungen aus.

let note = 2; if (note === 1) { console.log("Sehr gut!"); } else if (note === 2) { console.log("Gut!"); } else if (note === 3) { console.log("Befriedigend."); } else { console.log("Ausreichend oder schlechter."); }

Switch-Anweisung (switch)

Eine Alternative zu vielen else if-Blöcken, wenn eine Variable mit mehreren Werten verglichen wird.

let tag = "Montag"; switch (tag) { case "Montag": console.log("Wochenstart!"); break; // Wichtig, sonst "Fallthrough" case "Freitag": console.log("Fast Wochenende!"); break; default: // Optional, wenn kein case zutrifft console.log("Ein normaler Tag."); }

Schleifen

Wiederholen Codeblöcke.

for-Schleife

Wird verwendet, wenn die Anzahl der Wiederholungen bekannt ist.

for (let i = 0; i < 5; i++) { console.log("Durchlauf Nummer: " + i); }

while-Schleife

Wird verwendet, solange eine Bedingung wahr ist. Die Bedingung wird vor jeder Iteration geprüft.

let zaehler = 0; while (zaehler < 3) { console.log("Zähler: " + zaehler); zaehler++; }

do...while-Schleife

Ähnlich wie while, aber der Codeblock wird mindestens einmal ausgeführt, da die Bedingung erst nach der Iteration geprüft wird.

let j = 0; do { console.log("j ist: " + j); j++; } while (j < 0); // Wird trotzdem einmal ausgeführt, obwohl j nicht < 0 ist.

for...of-Schleife (ES6)

Iteriert über die Werte von iterierbaren Objekten (z.B. Arrays, Strings).

const farbenArray = ['rot', 'grün', 'blau']; for (const farbe of farbenArray) { console.log(farbe); }

for...in-Schleife

Iteriert über die Schlüssel (Eigenschaften) eines Objekts. Kann auch für Arrays verwendet werden, liefert dann aber die Indizes.

const auto = { marke: 'VW', modell: 'Golf', jahr: 2020 }; for (const eigenschaft in auto) { console.log(eigenschaft + ": " + auto[eigenschaft]); }

Funktionen

Funktionen sind wiederverwendbare Codeblöcke, die eine bestimmte Aufgabe erfüllen. Sie können Parameter entgegennehmen und einen Wert zurückgeben.

Funktionsdeklaration (Function Declaration)

function addiere(a, b) { return a + b; } let summe = addiere(5, 3); // summe ist 8 console.log(summe);

Funktionsausdruck (Function Expression)

Eine Funktion kann auch einer Variablen zugewiesen werden.

const multipliziere = function(a, b) { return a * b; }; let produkt = multipliziere(4, 2); // produkt ist 8 console.log(produkt);

Pfeilfunktionen (Arrow Functions, ES6)

Eine kürzere Syntax für Funktionsausdrücke. Sie haben auch ein anderes Verhalten bezüglich this.

const subtrahiere = (a, b) => a - b; // Kurzform, wenn nur ein Ausdruck zurückgegeben wird let differenz = subtrahiere(10, 7); // differenz ist 3 const gruss = name => console.log(`Hallo, ${name}!`); // Ein Parameter, keine Klammern nötig gruss('Peter'); const komplexeFunktion = (a, b) => { // Mehrere Anweisungen, geschweifte Klammern und 'return' nötig const resultat = a + b; return resultat * 2; };
+

Parameter und Argumente: Parameter sind die Namen, die in der Funktionsdefinition aufgelistet sind. Argumente sind die tatsächlichen Werte, die beim Aufruf der Funktion übergeben werden.

Default-Parameter (ES6): Ermöglicht das Setzen von Standardwerten für Parameter, falls keine Argumente übergeben werden.

function halloSagen(name = "Gast") { console.log(`Hallo, ${name}!`); } halloSagen("Lisa"); // Hallo, Lisa! halloSagen(); // Hallo, Gast!

Objekte

Objekte sind Sammlungen von Schlüssel-Wert-Paaren (Eigenschaften und Methoden). Eigenschaften sind Daten, Methoden sind Funktionen, die mit den Daten des Objekts arbeiten können.

Objektliteral

Der einfachste Weg, ein Objekt zu erstellen.

const meinAuto = { marke: "BMW", modell: "X5", farbe: "schwarz", baujahr: 2022, detailsAnzeigen: function() { // 'this' bezieht sich auf das aktuelle Objekt (meinAuto) console.log(`Marke: ${this.marke}, Modell: ${this.modell}, Baujahr: ${this.baujahr}`); }, // ES6 Kurzschreibweise für Methoden hupen() { console.log("Hup Hup!"); } }; // Zugriff auf Eigenschaften console.log(meinAuto.marke); // BMW (Dot Notation) console.log(meinAuto['modell']); // X5 (Bracket Notation - nützlich für dynamische Schlüssel) // Eigenschaft ändern meinAuto.farbe = "rot"; console.log(meinAuto.farbe); // rot // Methode aufrufen meinAuto.detailsAnzeigen(); meinAuto.hupen();

Konstruktor-Funktionen

Eine Vorlage zum Erstellen mehrerer Objekte desselben Typs.

function Auto(marke, modell, baujahr) { this.marke = marke; this.modell = modell; this.baujahr = baujahr; this.details = function() { console.log(`${this.marke} ${this.modell} (${this.baujahr})`); }; } const auto1 = new Auto('Audi', 'A4', 2021); const auto2 = new Auto('Mercedes', 'C-Klasse', 2023); auto1.details(); // Audi A4 (2021) auto2.details(); // Mercedes C-Klasse (2023)

Klassen (ES6)

Eine modernere Syntax für Konstruktor-Funktionen (syntaktischer Zucker über Prototypen).

class Fahrzeug { constructor(marke, modell) { this.marke = marke; this.modell = modell; } zeigeInfo() { console.log(`Fahrzeug: ${this.marke} ${this.modell}`); } } const meinFahrzeug = new Fahrzeug('Tesla', 'Model S'); meinFahrzeug.zeigeInfo(); // Fahrzeug: Tesla Model S class PKW extends Fahrzeug { // Vererbung constructor(marke, modell, tueren) { super(marke, modell); // Ruft den Konstruktor der Elternklasse auf this.tueren = tueren; } zeigeInfo() { // Methode überschreiben super.zeigeInfo(); console.log(`Anzahl Türen: ${this.tueren}`); } } const meinPKW = new PKW('VW', 'Golf', 5); meinPKW.zeigeInfo(); // Fahrzeug: VW Golf // Anzahl Türen: 5

Arrays

Arrays sind geordnete Listen von Werten. Ein Array kann Werte verschiedener Datentypen enthalten.

// Array-Literal const fruechte = ["Apfel", "Banane", "Orange"]; const gemischt = [1, "Text", true, { id: 7 }]; // Zugriff auf Elemente (Index beginnt bei 0) console.log(fruechte[0]); // "Apfel" console.log(fruechte[1]); // "Banane" // Element ändern fruechte[1] = "Birne"; console.log(fruechte); // ["Apfel", "Birne", "Orange"] // Länge des Arrays console.log(fruechte.length); // 3

Nützliche Array-Methoden

  • push(): Fügt ein Element am Ende hinzu.
  • pop(): Entfernt das letzte Element und gibt es zurück.
  • shift(): Entfernt das erste Element und gibt es zurück.
  • unshift(): Fügt ein Element am Anfang hinzu.
  • splice(index, anzahlEntfernen, element1, ...): Fügt Elemente hinzu/entfernt Elemente an einer bestimmten Position.
  • slice(startIndex, endIndex): Erstellt eine Kopie eines Teils des Arrays (endIndex nicht inklusive).
  • concat(array2, ...): Verbindet Arrays.
  • join(separator): Wandelt alle Elemente in einen String um, getrennt durch den Separator.
  • indexOf(element): Gibt den ersten Index eines Elements zurück (-1, wenn nicht gefunden).
  • includes(element) (ES7): Prüft, ob ein Element im Array enthalten ist (gibt true/false zurück).

Iterationsmethoden (ES5+):

  • forEach(callbackFn): Führt eine Funktion für jedes Element aus.
  • map(callbackFn): Erstellt ein neues Array mit den Ergebnissen der Callback-Funktion für jedes Element.
  • filter(callbackFn): Erstellt ein neues Array mit allen Elementen, für die die Callback-Funktion true zurückgibt.
  • reduce(callbackFn, initialValue): Reduziert das Array auf einen einzelnen Wert.
  • find(callbackFn): Gibt das erste Element zurück, das die Bedingung der Callback-Funktion erfüllt.
  • findIndex(callbackFn): Gibt den Index des ersten Elements zurück, das die Bedingung erfüllt.
  • some(callbackFn): Prüft, ob mindestens ein Element die Bedingung erfüllt.
  • every(callbackFn): Prüft, ob alle Elemente die Bedingung erfüllen.
const zahlen = [1, 2, 3, 4, 5]; zahlen.push(6); // [1, 2, 3, 4, 5, 6] zahlen.pop(); // [1, 2, 3, 4, 5] zahlen.forEach(function(zahl) { console.log(zahl * 2); }); // Gibt 2, 4, 6, 8, 10 aus const quadrate = zahlen.map(zahl => zahl * zahl); console.log(quadrate); // [1, 4, 9, 16, 25] const geradeZahlen = zahlen.filter(zahl => zahl % 2 === 0); console.log(geradeZahlen); // [2, 4]

DOM-Manipulation

Das Document Object Model (DOM) ist eine Programmierschnittstelle für HTML- und XML-Dokumente. Es repräsentiert die Seitenstruktur als Baum von Objekten (Nodes), sodass JavaScript darauf zugreifen und den Inhalt, die Struktur und das Styling der Seite dynamisch verändern kann.

Elemente auswählen

  • document.getElementById('id'): Wählt ein Element anhand seiner ID aus (schnellste Methode).
  • document.getElementsByTagName('tagname'): Gibt eine HTMLCollection aller Elemente mit dem angegebenen Tag-Namen zurück.
  • document.getElementsByClassName('classname'): Gibt eine HTMLCollection aller Elemente mit der angegebenen Klasse zurück.
  • document.querySelector('css-selector'): Gibt das erste Element zurück, das auf den CSS-Selektor passt.
  • document.querySelectorAll('css-selector'): Gibt eine NodeList aller Elemente zurück, die auf den CSS-Selektor passen.
<div id="meinDiv" class="box">Hallo</div> const divById = document.getElementById('meinDiv'); const divsByTag = document.getElementsByTagName('div'); // HTMLCollection const boxesByClass = document.getElementsByClassName('box'); // HTMLCollection const erstesDiv = document.querySelector('div.box'); // Erstes Element const alleDivs = document.querySelectorAll('div'); // NodeList console.log(divById.innerHTML); // "Hallo"

Inhalt und Attribute ändern

  • element.innerHTML: Liest oder setzt den HTML-Inhalt eines Elements. (Vorsicht bei Benutzereingaben wegen XSS-Gefahr).
  • element.textContent: Liest oder setzt den Textinhalt eines Elements (sicherer als innerHTML für reinen Text).
  • element.setAttribute('attributname', 'wert'): Setzt den Wert eines Attributs.
  • element.getAttribute('attributname'): Liest den Wert eines Attributs.
  • element.removeAttribute('attributname'): Entfernt ein Attribut.
  • element.style.eigenschaft = 'wert': Ändert CSS-Stile (z.B. element.style.color = 'red').
  • element.classList.add('klasse'), remove('klasse'), toggle('klasse'), contains('klasse'): Manipuliert CSS-Klassen.
const meinElement = document.getElementById('meinDiv'); if (meinElement) { meinElement.innerHTML = 'Neuer HTML Inhalt!'; meinElement.style.backgroundColor = 'yellow'; meinElement.setAttribute('data-custom', '123'); meinElement.classList.add('hervorgehoben'); }

Elemente erstellen und hinzufügen/entfernen

  • document.createElement('tagname'): Erstellt ein neues HTML-Element.
  • element.appendChild(kindElement): Fügt ein Kindelement am Ende eines Elements hinzu.
  • element.insertBefore(neuesElement, referenzElement): Fügt ein neues Element vor einem Referenz-Kindelement ein.
  • element.removeChild(kindElement): Entfernt ein Kindelement.
  • element.replaceChild(neuesKind, altesKind): Ersetzt ein Kindelement durch ein anderes.
const neuerP = document.createElement('p'); neuerP.textContent = 'Dies ist ein neuer Paragraph.'; document.body.appendChild(neuerP); // Fügt den Paragraphen am Ende des Body hinzu

    Moderne JavaScript-Features (ES6+)

    ECMAScript ist der Standard, auf dem JavaScript basiert. ECMAScript 2015 (ES6) brachte viele wichtige Neuerungen. Seitdem gibt es jährliche Updates.

    • let und const: Wurden bereits bei Variablen behandelt.
    • Pfeilfunktionen (Arrow Functions): Siehe Funktionen.
    • Template Literals (Template Strings): Erlauben eingebettete Ausdrücke und mehrzeilige Strings mit Backticks (`).
      const name = "Welt"; const alter = 10; const gruss = `Hallo ${name}! Du bist ${alter} Jahre alt.`; console.log(gruss);
    • Default-Parameter: Siehe Funktionen.
    • Rest-Parameter (...): Fasst "restliche" Funktionsargumente zu einem Array zusammen.
      function summiereAlle(...zahlen) { // zahlen ist ein Array return zahlen.reduce((summe, zahl) => summe + zahl, 0); } console.log(summiereAlle(1, 2, 3, 4)); // 10
    • Spread-Operator (...): Erweitert ein Array oder Objekt an Stellen, wo mehrere Elemente/Eigenschaften erwartet werden.
      const arr1 = [1, 2]; const arr2 = [...arr1, 3, 4]; // [1, 2, 3, 4] console.log(arr2); const obj1 = { a: 1, b: 2 }; const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 } console.log(obj2);
    • Destrukturierung (Destructuring Assignment): Ermöglicht das Extrahieren von Werten aus Arrays oder Eigenschaften aus Objekten in separate Variablen.
      // Array Destrukturierung const [erster, zweiter] = ["Apfel", "Banane"]; console.log(erster); // "Apfel" // Objekt Destrukturierung const person = { vorname: "Max", nachname: "Mustermann" }; const { vorname, nachname } = person; console.log(`${vorname} ${nachname}`); // "Max Mustermann" // Mit Umbenennung const { vorname: vName, nachname: nName } = person; console.log(`${vName} ${nName}`);
    • Klassen: Siehe Objekte.
    • Promises und Async/Await: Für asynchrone Programmierung (z.B. Daten von einem Server laden).
      // Promise Beispiel function ladeDaten() { return new Promise((resolve, reject) => { // Simuliere eine Serveranfrage setTimeout(() => { const erfolg = true; // Ändere dies zu false, um den reject-Fall zu testen if (erfolg) { resolve("Daten erfolgreich geladen!"); } else { reject("Fehler beim Laden der Daten."); } }, 1000); }); } ladeDaten() .then(nachricht => console.log(nachricht)) .catch(fehler => console.error(fehler)); // Async/Await Beispiel (syntaktischer Zucker für Promises) async function holeUndZeigeDaten() { try { const daten = await ladeDaten(); // Wartet, bis das Promise resolved oder rejected wird console.log("Async/Await: " + daten); } catch (error) { console.error("Async/Await Fehler: " + error); } } holeUndZeigeDaten();
    • Module (Import/Export): Ermöglichen das Aufteilen von Code in wiederverwendbare Module.
      // In datei "helfer.js": // export const PI = 3.14159; // export function add(a, b) { return a + b; } // In anderer Datei "main.js": // import { PI, add } from './helfer.js'; // console.log(PI); // console.log(add(2, 3)); Hinweis: Module funktionieren im Browser oft nur mit speziellen Setups (z.B. Bundler wie Webpack/Parcel oder type="module" im script-Tag).

    Fehlerbehandlung (Error Handling)

    Fehler können in JavaScript auftreten. Es ist wichtig, sie abzufangen und angemessen darauf zu reagieren, um Abstürze des Skripts zu verhindern.

    try...catch...finally

    • try: Umschließt den Code, der möglicherweise einen Fehler auslöst.
    • catch (error): Fängt den Fehler ab, falls einer im try-Block auftritt. Das error-Objekt enthält Informationen zum Fehler (z.B. error.message, error.name).
    • finally (optional): Code in diesem Block wird immer ausgeführt, egal ob ein Fehler aufgetreten ist oder nicht (z.B. zum Aufräumen).
    try { console.log("Versuche etwas..."); // nichtDefinierteFunktion(); // Dies würde einen Fehler auslösen let ergebnis = 10 / 0; // Ergibt Infinity, kein direkter Fehler hier, aber z.B. JSON.parse("ungültig") würde Fehler werfen if (ergebnis === Infinity) { throw new Error("Division durch Null ist nicht erlaubt in diesem Kontext!"); // Eigenen Fehler werfen } console.log("Das hat geklappt!"); } catch (fehler) { console.error("Ein Fehler ist aufgetreten!"); error("Fehlermeldung: " + fehler.message); error("Fehlertyp: " + fehler.name); // console.error(fehler.stack); // Gibt den Stack Trace aus } finally { log("Dieser Block wird immer ausgeführt."); }

    throw Anweisung

    Mit throw kann man eigene Fehler erstellen und werfen.

    function checkAge(alter) { if (alter < 18) { throw new Error("Zugriff verweigert: Du musst mindestens 18 Jahre alt sein."); } return "Zugriff gewährt."; } try { console.log(checkAge(16)); } catch (e) { error(e.message); // "Zugriff verweigert: Du musst mindestens 18 Jahre alt sein." }

    Weitere wichtige Konzepte

    • JSON (JavaScript Object Notation): Ein leichtgewichtiges Datenformat, das oft für den Datenaustausch zwischen Client und Server verwendet wird. Sieht aus wie JavaScript-Objektliterale.
      • JSON.stringify(objekt): Wandelt ein JavaScript-Objekt in einen JSON-String um.
      • JSON.parse(jsonString): Wandelt einen JSON-String in ein JavaScript-Objekt um.
      const datenObjekt = { name: "Test", wert: 123 }; const jsonString = JSON.stringify(datenObjekt); console.log(jsonString); // '{"name":"Test","wert":123}' const zurueckObjekt = JSON.parse(jsonString); console.log(zurueckObjekt.name); // "Test"
    • this-Schlüsselwort: Der Wert von this hängt davon ab, wie eine Funktion aufgerufen wird (Kontext). Bei Pfeilfunktionen erbt this vom umgebenden Kontext. Das Verhalten von this kann komplex sein und ist eine häufige Fehlerquelle.
    • Scope (Gültigkeitsbereich): Definiert, wo Variablen und Funktionen zugänglich sind (Global Scope, Function Scope, Block Scope).
    • Hoisting: Ein Mechanismus in JavaScript, bei dem Deklarationen von Variablen (mit var) und Funktionen an den Anfang ihres Scopes "gehoben" werden, bevor der Code ausgeführt wird. Zuweisungen bleiben an ihrer Stelle. let und const werden auch gehoistet, aber nicht initialisiert (Temporal Dead Zone).
    • Closures: Eine Funktion hat Zugriff auf Variablen aus ihrem äußeren (umschließenden) Scope, auch nachdem die äußere Funktion ihre Ausführung beendet hat.
    • Asynchrone Programmierung: Behandelt Operationen, die Zeit in Anspruch nehmen (z.B. API-Aufrufe, Timer), ohne den Hauptthread zu blockieren. Wichtige Konzepte sind Callbacks, Promises und Async/Await (siehe ES6+).
    • Strict Mode ("use strict";): Aktiviert einen strengeren Modus in JavaScript, der einige "stille" Fehler in echte Fehler umwandelt und einige unsichere Praktiken verbietet. Sollte am Anfang eines Skripts oder einer Funktion stehen.
      • "use strict"; // Aktiviert den Strict Mode für das gesamte Skript (wenn ganz oben) function meineFunktion() { // "use strict"; // Kann auch pro Funktion aktiviert werden // x = 3.14; // Fehler im Strict Mode (Variable nicht deklariert) let y = 10; console.log(y); } meineFunktion();
