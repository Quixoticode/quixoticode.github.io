Grundstruktur eines HTML-Dokuments

Jedes HTML-Dokument folgt einer grundlegenden Struktur, die dem Browser mitteilt, wie die Seite aufgebaut ist und interpretiert werden soll.

<!DOCTYPE html> <html lang="de"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Seitentitel</title> <!-- Hier können auch CSS-Stylesheets und JavaScript-Dateien verlinkt werden --> </head> <body> <!-- Der sichtbare Inhalt der Webseite kommt hier hin --> <h1>Hallo Welt!</h1> <p>Dies ist mein erstes HTML-Dokument.</p> </body> </html>

Erklärung der Teile:

  • <!DOCTYPE html>: Definiert den Dokumenttyp. Für HTML5 ist dies die Standarddeklaration.
  • <html lang="de">: Das Wurzelelement jeder HTML-Seite. Das lang-Attribut gibt die Sprache des Dokuments an (hier Deutsch).
  • <head>: Enthält Meta-Informationen über das HTML-Dokument, die nicht direkt auf der Seite angezeigt werden.
    • <meta charset="UTF-8">: Definiert die Zeichenkodierung, wichtig für die korrekte Darstellung von Sonderzeichen.
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Konfiguriert den Viewport für responsive Webdesign, sodass die Seite auf verschiedenen Gerätegrößen gut aussieht.
    • <title>Seitentitel</title>: Definiert den Titel der Seite, der im Browser-Tab oder Fensterbalken angezeigt wird.
  • <body>: Enthält den gesamten sichtbaren Inhalt der Webseite.

Textformatierung

HTML bietet verschiedene Elemente zur Strukturierung und Formatierung von Text.

Überschriften (<h1> bis <h6>)

Definieren Überschriften unterschiedlicher Ebenen. <h1> ist die wichtigste Überschrift, <h6> die unwichtigste.

Das ist eine H1 Überschrift

Das ist eine H2 Überschrift

Das ist eine H3 Überschrift

Das ist eine H4 Überschrift

Das ist eine H5 Überschrift
Das ist eine H6 Überschrift
<h1>Das ist eine H1 Überschrift</h1> <h2>Das ist eine H2 Überschrift</h2> <h3>Das ist eine H3 Überschrift</h3>

Absätze (<p>)

Definiert einen Textabsatz.

Dies ist ein Absatz. Er enthält Text, der zusammenhängend dargestellt wird.

Dies ist ein weiterer Absatz.

<p>Dies ist ein Absatz.</p> <p>Dies ist ein weiterer Absatz.</p>

Hervorhebungen

  • <strong> oder <b>: Für stark hervorgehobenen (fettgedruckten) Text. <strong> impliziert semantische Wichtigkeit.
  • <em> oder <i>: Für betonten (kursiven) Text. <em> impliziert semantische Betonung.
  • <u>: Für unterstrichenen Text (sparsam verwenden, da es oft mit Links verwechselt wird).
  • <s> oder <del>: Für durchgestrichenen Text (z.B. veraltete Information).
  • <mark>: Für markierten oder hervorgehobenen Text.
  • <small>: Für kleineren Text (z.B. Copyright-Hinweise).
  • <sup> und <sub>: Für hochgestellten und tiefgestellten Text.

Wichtig! Dieser Text ist kursiv und unterstrichen. Markiert! Kleiner Text. E = mc2. H2O.

<p><strong>Wichtig!</strong> Dieser Text ist <em>kursiv</em>.</p>

Zeilenumbruch (<br>)

Erzwingt einen Zeilenumbruch innerhalb eines Textblocks. Es ist ein leeres Element (kein schließendes Tag in HTML5, in XHTML <br />).

Dieser Text ist in der ersten Zeile.
Und dieser Text ist in der zweiten Zeile.
Dieser Text ist in der ersten Zeile.<br> Und dieser Text ist in der zweiten Zeile.

Horizontale Linie (<hr>)

Erzeugt eine horizontale Linie, die oft zur thematischen Trennung von Inhalten verwendet wird. Auch ein leeres Element.

Inhalt oberhalb der Linie.

Inhalt unterhalb der Linie.

<p>Inhalt oberhalb der Linie.</p> <hr> <p>Inhalt unterhalb der Linie.</p>

Vorformatierter Text (<pre>)

Stellt Text so dar, wie er im Quellcode formatiert ist, inklusive Leerzeichen und Zeilenumbrüchen. Oft für Code-Beispiele verwendet.

function hallo() {
  console.log("Hallo Welt!");
}
<pre> function hallo() { console.log("Hallo Welt!"); } </pre>

Code (<code>)

Markiert ein Stück Computercode. Wird oft innerhalb von <pre> für Codeblöcke oder inline in Absätzen verwendet.

Die Funktion alert() zeigt eine Nachricht an.

<p>Die Funktion <code>alert()</code> zeigt eine Nachricht an.</p>

Zitate

  • <blockquote>: Für längere Zitate, die als eigener Block dargestellt werden.
    Dies ist ein längeres Zitat, das von einer anderen Quelle stammt.
    <blockquote cite="quelle.url"> Zitattext </blockquote>
  • <q>: Für kurze, inline Zitate. Browser fügen oft automatisch Anführungszeichen hinzu.

    Er sagte: Hallo Welt.

    <p>Er sagte: <q>Hallo Welt</q>.</p>
  • <cite>: Um den Titel eines Werkes (z.B. Buch, Film) zu kennzeichnen.

    Mein Lieblingsbuch ist Der Herr der Ringe.

    <p>Mein Lieblingsbuch ist <cite>Der Herr der Ringe</cite>.</p>
  • <abbr>: Für Abkürzungen oder Akronyme. Das title-Attribut kann die volle Bedeutung anzeigen.

    HTML ist toll.

    <p><abbr title="Hypertext Markup Language">HTML</abbr> ist toll.</p>
  • <address>: Für Kontaktinformationen (Autor, Besitzer der Webseite).
    Geschrieben von Max Mustermann.
    Besuche uns auf:
    Beispielweg 123, Musterstadt
    Deutschland
    <address> Geschrieben von Max Mustermann.<br> Besuche uns auf:<br> Beispielweg 123, Musterstadt<br> Deutschland </address>

Listen

HTML bietet drei Haupttypen von Listen.

Ungeordnete Listen (<ul>)

Eine Liste von Elementen, bei der die Reihenfolge keine Rolle spielt. Wird typischerweise mit Aufzählungspunkten dargestellt.

  • Apfel
  • Banane
  • Orange
<ul> <li>Apfel</li> <li>Banane</li> <li>Orange</li> </ul>

Geordnete Listen (<ol>)

Eine Liste von Elementen, bei der die Reihenfolge wichtig ist. Wird typischerweise nummeriert dargestellt.

  1. Vorbereitung
  2. Durchführung
  3. Abschluss
<ol> <li>Vorbereitung</li> <li>Durchführung</li> <li>Abschluss</li> </ol>

Das Attribut type kann verwendet werden, um den Nummerierungsstil zu ändern (z.B. type="A" für Großbuchstaben, type="i" für kleine römische Zahlen). Das Attribut start kann den Startwert der Nummerierung festlegen.

Beschreibungslisten (<dl>)

Eine Liste von Begriffen und deren Beschreibungen.

  • <dl>: (Description List) Umschließt die gesamte Liste.
  • <dt>: (Description Term) Definiert den Begriff.
  • <dd>: (Description Details) Definiert die Beschreibung des Begriffs.
HTML
Hypertext Markup Language - Die Standardsprache zur Erstellung von Webseiten.
CSS
Cascading Style Sheets - Wird verwendet, um das Aussehen von HTML-Dokumenten zu gestalten.
<dl> <dt>HTML</dt> <dd>Hypertext Markup Language - Die Standardsprache zur Erstellung von Webseiten.</dd> <dt>CSS</dt> <dd>Cascading Style Sheets - Wird verwendet, um das Aussehen von HTML-Dokumenten zu gestalten.</dd> </dl>

Listen können auch verschachtelt werden, indem eine Liste innerhalb eines <li>-Elements platziert wird.

Links (<a> - Anchor)

Links sind das Herzstück des Internets und ermöglichen die Navigation zwischen Seiten und Ressourcen.

Das <a>-Element (Anchor) wird verwendet, um Links zu erstellen. Das wichtigste Attribut ist href (Hypertext Reference), das das Ziel des Links angibt.

Externe Links

Verlinken zu einer anderen Webseite.

Zu Google
<a href="https://www.google.com">Zu Google</a>

Interne Links (zu anderen Seiten der eigenen Website)

Verlinken zu einer anderen HTML-Datei auf derselben Website.

Zur anderen Seite
<a href="andere-seite.html">Zur anderen Seite</a>

Anker-Links (zu einem bestimmten Abschnitt auf derselben Seite)

Verlinken zu einem Element mit einer bestimmten id auf der aktuellen Seite.

Zurück zum Abschnitt Grundstruktur
<!-- Ziel des Links --> <h2 id="struktur">Grundstruktur</h2> <!-- Der Link selbst --> <a href="#struktur">Zum Abschnitt Grundstruktur</a>

E-Mail-Links

Öffnet das Standard-E-Mail-Programm des Benutzers mit einer voradressierten E-Mail.

Sende uns eine E-Mail
<a href="mailto:info@example.com">Sende uns eine E-Mail</a>

Telefon-Links

Ermöglicht das Anrufen einer Telefonnummer auf Geräten, die dies unterstützen.

Ruf uns an
<a href="tel:+49123456789">Ruf uns an</a>

Das target-Attribut

Bestimmt, wo der Link geöffnet wird:

  • _self: (Standard) Öffnet im selben Fenster/Tab.
  • _blank: Öffnet in einem neuen Fenster/Tab. Aus Sicherheitsgründen sollte hier oft rel="noopener noreferrer" hinzugefügt werden.
  • _parent: Öffnet im übergeordneten Frame (bei Framesets).
  • _top: Öffnet im obersten Frame (bei Framesets).
Wikipedia in neuem Tab öffnen
<a href="https://www.wikipedia.org" target="_blank" rel="noopener noreferrer">Wikipedia in neuem Tab öffnen</a>

Das title-Attribut

Gibt zusätzliche Informationen zum Link an, die oft als Tooltip angezeigt werden, wenn man mit der Maus darüberfährt.

Mozilla
<a href="https://www.mozilla.org" title="Zur Mozilla Homepage">Mozilla</a>

Bilder (<img>)

Das <img>-Element wird verwendet, um Bilder in eine Webseite einzubetten. Es ist ein leeres Element.

Wichtige Attribute:

  • src (Source): Gibt den Pfad zur Bilddatei an (URL oder lokaler Pfad). **Erforderlich.**
  • alt (Alternative Text): Beschreibt das Bild. Wichtig für Barrierefreiheit (Screenreader) und falls das Bild nicht geladen werden kann. **Erforderlich.**
  • width: Definiert die Breite des Bildes in Pixeln.
  • height: Definiert die Höhe des Bildes in Pixeln.

    Hinweis: Es ist besser, die Größe von Bildern mit CSS zu steuern, um Flexibilität und responsives Verhalten zu gewährleisten. Wenn width und height in HTML gesetzt werden, sollte das Seitenverhältnis beibehalten werden, um Verzerrungen zu vermeiden.

<img src="pfad/zum/bild.jpg" alt="Beschreibung des Bildes" width="300" height="150"> <!-- Responsives Bild mit CSS (Klasse .responsive oben im <style>-Tag definiert) --> <img src="pfad/zum/bild.jpg" alt="Beschreibung des Bildes" class="responsive">

Bilder als Links

Ein Bild kann auch als Link fungieren, indem das <img>-Element in ein <a>-Element eingebettet wird.

<a href="ziel-url.html"> <img src="bild.png" alt="Link-Beschreibung"> </a>

Das <figure> und <figcaption> Element

<figure> wird verwendet, um eigenständige Inhalte wie Diagramme, Illustrationen, Fotos, Code-Listings usw. zu umschließen. <figcaption> kann verwendet werden, um eine Beschriftung für das <figure>-Element bereitzustellen.

<figure> <img src="bild.jpg" alt="Beschreibung"> <figcaption>Abb. 1: Bildunterschrift.</figcaption> </figure>

Tabellen

Tabellen werden verwendet, um tabellarische Daten darzustellen.

  • <table>: Umschließt die gesamte Tabelle.
  • <tr> (Table Row): Definiert eine Zeile in der Tabelle.
  • <th> (Table Header): Definiert eine Kopfzelle. Der Text ist standardmäßig fett und zentriert.
  • <td> (Table Data): Definiert eine Standardzelle (Datenzelle).
  • <caption>: Fügt eine Beschriftung zur Tabelle hinzu (sollte das erste Kindelement von <table> sein).
  • <thead>: Gruppiert die Kopfzeilen der Tabelle.
  • <tbody>: Gruppiert den Hauptinhalt (Körper) der Tabelle.
  • <tfoot>: Gruppiert die Fußzeilen der Tabelle.

Wichtiger Hinweis: Tabellen sollten nur für tabellarische Daten verwendet werden, nicht für das Layout von Webseiten. Für Layoutzwecke gibt es modernere Techniken wie CSS Flexbox und Grid.

Monatliche Ausgaben
Kategorie Betrag (€)
Miete 800
Lebensmittel 300
Transport 100
Gesamt 1200
<table> <caption>Monatliche Ausgaben</caption> <thead> <tr> <th>Kategorie</th> <th>Betrag (€)</th> </tr> </thead> <tbody> <tr> <td>Miete</td> <td>800</td> </tr> <tr> <td>Lebensmittel</td> <td>300</td> </tr> </tbody> <tfoot> <tr> <td>Gesamt</td> <td>1100</td> <!-- Korrigierter Wert für das Beispiel --> </tr> </tfoot> </table>

Attribute colspan und rowspan

colspan: Lässt eine Zelle sich über mehrere Spalten erstrecken.

rowspan: Lässt eine Zelle sich über mehrere Zeilen erstrecken.

Name Max Mustermann
Telefon Privat 0123-45678
Mobil 0987-65432
<table> <tr> <th>Name</th> <td colspan="2">Max Mustermann</td> </tr> <tr> <td rowspan="2">Telefon</td> <td>Privat</td> <td>0123-45678</td> </tr> <tr> <td>Mobil</td> <td>0987-65432</td> </tr> </table>

Formulare (<form>)

Formulare werden verwendet, um Benutzereingaben zu sammeln.

Das <form>-Element umschließt die verschiedenen Formularfelder. Wichtige Attribute sind:

  • action: Die URL, an die die Formulardaten gesendet werden sollen (z.B. ein serverseitiges Skript).
  • method: Die HTTP-Methode, mit der die Daten gesendet werden (üblicherweise GET oder POST).
    • GET: Daten werden an die URL angehängt (sichtbar, begrenzt).
    • POST: Daten werden im HTTP-Body gesendet (nicht sichtbar, für größere/sensible Daten).

Beschriftungen (<label>)

Das <label>-Element ordnet einem Formularfeld eine Beschriftung zu. Das for-Attribut des Labels sollte mit der id des zugehörigen Formularfelds übereinstimmen, um die Barrierefreiheit zu verbessern (Klick auf Label fokussiert das Feld).

Weitere gängige type-Attribute für <input>:

  • text: Einzeiliges Textfeld (Standard).
  • password: Für Passwörter (Zeichen werden maskiert).
  • email: Für E-Mail-Adressen (mit Validierung).
  • number: Für Zahlen (mit Pfeilen zum Erhöhen/Verringern).
  • date: Für Datumsauswahl. Weitere: month, week, time, datetime-local.
  • radio: Für eine Auswahl aus mehreren Optionen, bei der nur eine gewählt werden kann (gleicher name-Wert erforderlich).
  • checkbox: Für Optionen, bei denen mehrere ausgewählt werden können.
  • color: Für Farbauswahl.
  • range: Für einen Schieberegler.
  • file: Für Dateiupload.
  • submit: Sendet das Formular ab.
  • reset: Setzt alle Formularfelder auf ihre Standardwerte zurück.
  • button: Ein klickbarer Button (oft mit JavaScript verwendet).
  • hidden: Ein verstecktes Feld, dessen Wert mitgesendet wird.
  • search: Für Suchfelder.
  • tel: Für Telefonnummern.
  • url: Für URLs.

Wichtige Attribute für <input>:

  • name: Notwendig, damit die Daten des Feldes gesendet werden.
  • id: Eindeutiger Identifikator, wird für <label> verwendet.
  • value: Der initiale Wert des Feldes (oder der gesendete Wert bei Radio/Checkbox/Submit).
  • placeholder: Ein Hinweistext, der im Feld angezeigt wird, solange es leer ist.
  • required: Gibt an, dass das Feld ausgefüllt werden muss.
  • disabled: Deaktiviert das Feld.
  • readonly: Der Wert kann nicht geändert werden, wird aber gesendet.
  • checked: Wählt eine Checkbox oder einen Radio-Button standardmäßig aus.
  • min, max, step: Für numerische Eingaben oder Datums-/Zeiteingaben.
  • maxlength: Maximale Anzahl an Zeichen für Textfelder.
  • pattern: Ein regulärer Ausdruck zur Validierung der Eingabe.

Textbereich (<textarea>)

Für mehrzeilige Texteingaben.



<label for="comment">Kommentar:</label> <textarea id="comment" name="user_comment" rows="4" cols="50" placeholder="Ihr Kommentar..."></textarea>

Auswahllisten (<select>, <option>, <optgroup>)

Erstellt ein Dropdown-Menü oder eine Auswahlliste.

  • <select>: Umschließt die Optionen.
    • name: Name des Select-Elements.
    • multiple: Erlaubt die Auswahl mehrerer Optionen (Liste statt Dropdown).
    • size: Anzahl der sichtbaren Optionen (bei multiple oder wenn größer 1).
  • <option>: Definiert eine einzelne Auswahlmöglichkeit.
    • value: Der Wert, der gesendet wird, wenn diese Option ausgewählt ist.
    • selected: Wählt diese Option standardmäßig aus.
  • <optgroup>: Gruppiert zusammengehörige Optionen.
    • label: Die Beschriftung für die Gruppe.





<label for="cars">Wähle ein Auto:</label> <select id="cars" name="car_choice"> <option value="volvo">Volvo</option> <option value="saab">Saab</option> <option value="mercedes" selected>Mercedes</option> <!-- vorausgewählt --> <option value="audi">Audi</option> </select>

Button-Element (<button>)

Eine flexiblere Alternative zu <input type="button"> oder <input type="submit">, da es auch HTML-Inhalt (z.B. Bilder, Textformatierung) enthalten kann.

Das type-Attribut kann sein: submit (Standard), reset oder button.

<button type="submit">Formular <strong>Absenden</strong></button> <button type="button" onclick="doSomething()">Klick mich!</button>

  • <fieldset>: Gruppiert zusammengehörige Elemente in einem Formular.
  • <legend>: Definiert eine Überschrift für ein <fieldset>.
  • <datalist>: Bietet eine Liste von vordefinierten Optionen für ein <input>-Feld (Autovervollständigung).

  • <output>: Zeigt das Ergebnis einer Berechnung oder Benutzeraktion an.
  • <progress>: Zeigt den Fortschritt einer Aufgabe an.
  • <meter>: Stellt einen skalaren Messwert innerhalb eines bekannten Bereichs dar (z.B. Speicherplatzbelegung).

Semantisches HTML

Semantische HTML-Elemente beschreiben die Bedeutung oder den Zweck ihres Inhalts, anstatt nur dessen Aussehen. Dies verbessert die Barrierefreiheit, SEO (Suchmaschinenoptimierung) und die Lesbarkeit des Codes.

Viele dieser Elemente wurden mit HTML5 eingeführt.

Strukturelle Elemente

  • <header>: Definiert einen Kopfbereich für ein Dokument oder einen Abschnitt. Enthält oft Überschriften, Logos, Navigationslinks.
  • <nav>: Definiert einen Bereich mit Navigationslinks.
  • <main>: Definiert den Hauptinhalt eines Dokuments. Es sollte nur ein <main>-Element pro Seite geben und es sollte nicht innerhalb von <article>, <aside>, <footer>, <header> oder <nav> verschachtelt sein.
  • <article>: Definiert einen in sich geschlossenen, unabhängigen Inhalt (z.B. Blogbeitrag, Nachrichtenartikel, Forumspost).
  • <section>: Definiert einen thematischen Abschnitt innerhalb eines Dokuments. Hat typischerweise eine eigene Überschrift.
  • <aside>: Definiert Inhalt, der nur lose mit dem umgebenden Inhalt in Beziehung steht (z.B. Seitenleiste, Glossar, Werbung).
  • <footer>: Definiert einen Fußbereich für ein Dokument oder einen Abschnitt. Enthält oft Copyright-Informationen, Autor, Links zu verwandten Dokumenten.
  • <address>: Definiert Kontaktinformationen für den Autor/Besitzer eines Dokuments oder Artikels.

Meine Webseite (<header>)

<nav>: Home | Über uns | Kontakt

<main>: Hauptinhalt der Seite.

<article>: Titel eines Artikels

Dies ist der Inhalt des Artikels...

<section>: Ein Abschnitt im Artikel

Details zu diesem Abschnitt.

<aside>: Seitenleisteninhalt.

<footer>: © 2025 Meine Webseite

<body> <header> <h1>Website Titel</h1> <nav> <ul> <li><a href="#">Home</a></li> <li><a href="#">Blog</a></li> </ul> </nav> </header> <main> <article> <h2>Titel des Blogbeitrags</h2> <p>Inhalt des Beitrags...</p> <section> <h3>Unterabschnitt</h3> <p>Details zum Unterabschnitt...</p> </section> </article> <aside> <h3>Verwandte Links</h3> <ul> <li><a href="#">Link 1</a></li> </ul> </aside> </main> <footer> <p>© 2025 Meine Webseite</p> <address> Kontakt: <a href="mailto:info@example.com">info@example.com</a> </address> </footer> </body>

Früher wurden oft <div>-Elemente mit IDs oder Klassen für diese Zwecke verwendet (z.B. <div id="header">). Die semantischen Elemente machen den Code verständlicher und strukturierter.

Weitere semantische Elemente

  • <figure> und <figcaption>: Siehe Abschnitt Bilder.
  • <time>: Definiert ein bestimmtes Datum oder eine bestimmte Uhrzeit. Das datetime-Attribut gibt das maschinenlesbare Format an.

    Veröffentlicht am 11. Mai 2025.

    Das Meeting ist um 14:00 Uhr.

    <p>Veröffentlicht am <time datetime="2025-05-11">11. Mai 2025</time>.</p>
  • <mark>: Text, der als markiert oder hervorgehoben dargestellt werden soll.
  • <details> und <summary>: Erstellen ein aufklappbares Widget. <summary> ist die sichtbare Überschrift.
    Klicken Sie hier für Details

    Dies sind die versteckten Details, die nach einem Klick auf die Zusammenfassung sichtbar werden.

    <details> <summary>Copyright Informationen</summary> <p>- Alle Rechte vorbehalten - </p> </details>

Multimedia (Audio & Video)

HTML5 bietet native Unterstützung für das Einbetten von Audio- und Videoinhalten ohne Plugins.

Audio (<audio>)

Bettet Audioinhalte ein.

  • src: Pfad zur Audiodatei.
  • controls: Zeigt Standard-Steuerelemente an (Play, Pause, Lautstärke etc.).
  • autoplay: Spielt die Audiodatei automatisch ab (oft von Browsern blockiert oder eingeschränkt).
  • loop: Spielt die Audiodatei in einer Schleife ab.
  • muted: Stummschaltung der Audiodatei.
  • preload: Gibt an, wie die Audiodatei beim Laden der Seite geladen werden soll (auto, metadata, none).

Das <source>-Element kann innerhalb von <audio> verwendet werden, um mehrere Audioformate anzubieten, falls ein Browser ein bestimmtes Format nicht unterstützt.

Hinweis: Da keine echte Audiodatei "beispiel.mp3" verlinkt ist, werden die Controls möglicherweise nicht korrekt oder gar nicht angezeigt.

<audio controls> <source src="audiofile.mp3" type="audio/mpeg"> <source src="audiofile.ogg" type="audio/ogg"> Ihr Browser unterstützt das Audio-Element nicht. </audio>

Video (<video>)

Bettet Videoinhalte ein. Hat ähnliche Attribute wie <audio>.

  • src: Pfad zur Videodatei.
  • controls: Zeigt Standard-Steuerelemente an.
  • autoplay: Spielt das Video automatisch ab (oft blockiert/eingeschränkt, oft nur mit muted erlaubt).
  • loop: Spielt das Video in einer Schleife ab.
  • muted: Stummschaltung des Videos.
  • poster: Ein Bild, das angezeigt wird, während das Video lädt oder bis der Benutzer auf Play klickt.
  • width und height: Dimensionen des Videoplayers.
  • preload: (auto, metadata, none).

Auch hier kann das <source>-Element für verschiedene Videoformate (z.B. MP4, WebM, Ogg) verwendet werden.

Hinweis: Da keine echte Videodatei "beispiel.mp4" verlinkt ist, wird nur das Posterbild bzw. eine Fehlermeldung angezeigt.

<video controls width="640" height="360" poster="posterbild.jpg"> <source src="videofile.mp4" type="video/mp4"> <source src="videofile.webm" type="video/webm"> Ihr Browser unterstützt das Video-Element nicht. </video>

Einbetten von externen Inhalten (<iframe>)

Das <iframe> (Inline Frame) Element wird verwendet, um ein anderes HTML-Dokument innerhalb der aktuellen Seite einzubetten. Dies wird oft für YouTube-Videos, Google Maps Karten oder andere externe Widgets genutzt.

  • src: Die URL des einzubettenden Dokuments.
  • width und height: Breite und Höhe des Frames.
  • frameborder: (veraltet, besser mit CSS) Ob ein Rahmen angezeigt wird (0 oder 1).
  • allowfullscreen: Erlaubt den Vollbildmodus für den Inhalt des iframes (z.B. bei Videos).
  • sandbox: Aktiviert Sicherheitseinschränkungen für den iframe-Inhalt.

Beispiel: Einbetten einer Webseite (aus Sicherheitsgründen kann nicht jede Seite eingebettet werden):

Beispiel: Einbetten eines YouTube-Videos (Platzhalter):

<!-- Einbetten einer Webseite --> <iframe src="https://www.wikipedia.org" width="600" height="400"></iframe> <!-- Einbetten eines YouTube Videos (Beispiel-Code von YouTube) --> <iframe width="560" height="315" src="https://www.youtube.com/embed/dQw4w9WgXcQ" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>

Sonstige wichtige Elemente und Konzepte

Kommentare

Kommentare werden vom Browser ignoriert und dienen dazu, den HTML-Code für Entwickler verständlicher zu machen.

<!-- Dies ist ein Kommentar. Er wird nicht auf der Webseite angezeigt. --> <p>Dieser Text wird angezeigt.</p> <!-- Mehrzeilige Kommentare sind auch möglich. -->

Das <div>-Element

Ein generisches Container-Element, das keine eigene semantische Bedeutung hat. Es wird verwendet, um Inhalte zu gruppieren und zu strukturieren, oft für Styling-Zwecke mit CSS oder für Manipulation mit JavaScript.

Ein Div-Container

Dieser Inhalt ist in einem Div.

<div class="container"> <h1>Titel im Div</h1> <p>Absatz im Div.</p> </div>

Das <span>-Element

Ein generisches Inline-Container-Element, das ebenfalls keine eigene semantische Bedeutung hat. Es wird verwendet, um Teile eines Textes oder Inline-Inhalts zu gruppieren, oft für Styling oder JavaScript.

Dies ist ein Satz mit einem roten und fetten Wort.

<p>Ein Teil dieses Textes ist <span class="hervorhebung">besonders</span> formatiert.</p>

Globale Attribute

Es gibt einige Attribute, die für die meisten HTML-Elemente verwendet werden können:

  • id: Definiert einen eindeutigen Identifikator für ein Element (darf auf einer Seite nur einmal vorkommen).
  • class: Weist einem Element eine oder mehrere Klassen zu (durch Leerzeichen getrennt). Klassen werden oft für CSS-Styling oder JavaScript-Selektion verwendet.
  • style: Ermöglicht das direkte Anwenden von Inline-CSS-Stilen auf ein Element (sollte sparsam verwendet werden; externe oder interne Stylesheets sind meist besser).
  • title: Zeigt zusätzliche Informationen als Tooltip an, wenn man mit der Maus über das Element fährt.
  • lang: Gibt die Sprache des Elementinhalts an.
  • tabindex: Steuert die Tabulator-Reihenfolge für die Tastaturnavigation und ob ein Element fokussierbar ist.
  • hidden: Blendet ein Element aus.
  • data-*: Benutzerdefinierte Datenattribute, um private Daten zu speichern, die von JavaScript genutzt werden können (z.B. data-user-id="123").
<div id="main-content" class="content-area important" style="color: blue;" title="Hauptinhaltsbereich"> <p lang="en" data-info="Ein Beispielabsatz">This is some content.</p> </div>

HTML-Entitäten

Bestimmte Zeichen haben in HTML eine spezielle Bedeutung (z.B. <, >, &). Um diese Zeichen als Text darzustellen, müssen sie als Entitäten geschrieben werden:

  • &lt; für < (kleiner als)
  • &gt; für > (größer als)
  • &amp; für & (Ampersand)
  • &quot; für " (doppeltes Anführungszeichen)
  • &apos; für ' (einfaches Anführungszeichen - nicht immer nötig, aber sicherer)
  • &nbsp; für ein geschütztes Leerzeichen (non-breaking space)
  • &copy; für © (Copyright)
  • &reg; für ® (Registered Trademark)
  • &euro; für € (Eurozeichen)

Der Code für ein Paragraphen-Tag ist &lt;p&gt;.

<p>Das &lt;p&gt;-Tag wird für Absätze verwendet.</p> <p>Kosten: 10&nbsp;&euro;</p>
